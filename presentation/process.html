Authentication Process

Процесс аутентификации можно разбить на несколько шагов, в зависимости от используемого метода:

Шаг 1: Пользователь предоставляет учетные данные

Пользователь отправляет свои учетные данные, такие как имя пользователя и пароль. В системах с федеративной аутентификацией это может включать отправку учетных данных внешнему поставщику идентификации (IdP).

Шаг 2: Система проверяет учетные данные

Система или внешний IdP проверяет предоставленные учетные данные. Для токенов, таких как JWT, IdP выдает токен после успешной проверки.

Шаг 3: Выпуск токена или создание сессии

Если учетные данные правильны, система генерирует токен аутентификации или сессию.

Шаг 4: Включение токена в запросы

Пользователь включает токен аутентификации (или cookie сессии) в последующие HTTP-запросы для доступа к ресурсам.

Шаг 5: Проверка токена/сессии при каждом запросе

Каждый раз, когда пользователь отправляет запрос, система проверяет токен или сессию для подтверждения аутентификации.

Шаг 6: Авторизация

После проверки токена система проверяет претензии (claims) или данные сессии для определения того, какие действия может выполнять пользователь.
